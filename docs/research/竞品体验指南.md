## 🚀 快速开始

### Step 1: 复制模板
为每个竞品app复制一份模板：
```bash
cp 竞品体验_模板.md 竞品体验报告/竞品体验_[App名称].md
```

### Step 2: 体验app
下载并体验竞品app，同时填写报告

### Step 3: 保存截图
将关键页面截图保存到 `screenshots/[App名称]/` 目录

### Step 4: 完成报告
填写完所有内容后，提交到 `竞品体验报告/` 目录

## 📸 截图规范

### 必须截取的页面
每个app至少截取6个关键页面：

1. **01_home.png** - 首页/启动页
2. **02_checkin.png** - 打卡界面
3. **03_timeline.png** - 时间线/历史记录
4. **04_stats.png** - 个人中心/数据统计
5. **05_pricing.png** - 付费界面
6. **06_settings.png** - 设置页面

### 截图命名规范
```
[序号]_[页面名称].png

示例：
01_home.png
02_checkin.png
03_timeline.png
```

### 截图要求
- 清晰完整，不要截取局部
- 包含状态栏（显示时间、电量）
- 不要包含个人隐私信息
- 建议使用iPhone/Android原生的截图功能

## ✅ 填写要点

### 基本信息
- **准确填写**App Store评分、价格等客观数据
- **截图证明**定价信息

### 产品定位
- **客观描述**这个app的定位，不要主观臆断
- **一句话总结**它的核心价值

### 核心功能
- **逐一体验**每个功能，不要遗漏
- **评分说明**给1-5星评分，并说明理由

### UI/UX设计
- **重点分析**是否符合"简单"理念
- **记录3个亮点**和3个槽点

### 商业模式
- **清晰划分**免费和付费功能
- **分析定价**是否合理

### 优缺点分析
- **具体举例**，不要泛泛而谈
- **说明原因**，不要只说"好/坏"

### 可借鉴点
- **详细说明**为什么好，如何借鉴
- **结合自己的产品**，思考差异化

## 🎯 体验时的核心问题

时刻记住你的产品定义：
- ✅ **不是**瑜伽学习app
- ✅ **不是**体式教学app
- ✅ **就是**专注打卡 + 身体觉察的记录工具

### 重点观察
1. **打卡流程**是否简单？
2. **时间线展示**是否美观？
3. **数据统计**是否有价值？
4. **UI设计**是否符合你的审美？
5. **定价模式**用户是否接受？

### 思考差异
1. 我可以做得更简单吗？
2. 我可以做得更美观吗？
3. 我可以做得更有仪式感吗？
4. 我的独特价值是什么？

## 💡 提示与技巧

### 1. 先体验后记录
- 不要边体验边写
- 先完整体验一遍，再集中填写

### 2. 截图即笔记
- 截图时记录关键想法
- 用截图标注功能标注关键点

### 3. 对比思考
- 体验多个app后对比
- 找出共性和差异

### 4. 用户体验
- 从用户视角思考
- 不要从开发者视角

### 5. 记录灵感
- 随时记录想法
- 不要怕想法"不成熟"
