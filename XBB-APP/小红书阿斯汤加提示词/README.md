# 小红书阿斯汤加内容生产系统

从碎片化创作升级到系统化创作。

---

## 🎯 核心理念

**❌ 碎片化创作**：
有想法 → 问AI → 得到答案 → 发布 → 忘记

**✅ 系统化创作**：
有想法 → 记录选题 → 检索素材 → 复用框架 → 发布 → 数据复盘 → 反哺方法论

每次创作都在给系统添砖加瓦。

---

## 🚀 快速开始

### 第一次使用

1. **阅读系统指南**：查看 `CLAUDE.md`
2. **了解素材库**：浏览 `内容素材库/` 目录
3. **记录第一个选题**：对 Claude 说"记录选题"

### 日常使用流程

```
[记录选题] → [深化选题] → [生成标题] → [发布] → [数据复盘]
```

## 🤖 AI 自动化能力

### 6大核心能力

1. **记录选题** 📝
   - 触发词："记录选题"
   - 动作：将想法记录到选题收集箱

2. **深化选题** ✍️
   - 触发词："深化选题"
   - 动作：检索素材库 → 生成文稿 → 保存到待深化

3. **生成标题** 📌
   - 触发词："生成标题"
   - 动作：基于文稿生成3个标题选项

4. **数据复盘** 📊
   - 触发词："数据复盘"
   - 动作：记录数据 → 移动到已发布 → 反哺方法论

5. **检索素材** 🔍
   - 触发词："检索素材 [关键词]"
   - 动作：在素材库中搜索相关内容

6. **方法论沉淀** 💡
   - 触发词："方法论"
   - 动作：分析数据 → 总结规律 → 优化方法论

## 📊 数据驱动流程

### 发布后的动作

每次发布后，必须做"数据复盘"：

1. 记录数据（阅读、点赞、收藏、评论）
2. 文稿移动到"已发布"
3. AI 自动分析数据
4. 更新统计文件
5. 反哺方法论

### 每月复盘

- 每月查看 `内容数据统计/` 文件
- 分析什么选题、标题、文案角度数据好
- 更新 `方法论沉淀/` 文件

## ✅ 使用建议

### 要做的

1. **每次创作前，先检索素材库**
   - 避免重复造轮子
   - 复用已验证的框架

2. **数据好的内容，标记为可复用**
   - 添加到 `爆款文稿库/`
   - 下次优先复用

3. **每次发布后，记录数据**
   - 数据驱动迭代
   - 不凭感觉，凭数据

4. **定期复盘**
   - 每周/每月回顾方法论
   - 持续优化

### 不要做的

1. 不要每次都从零开始创作
2. 不要忽视数据记录
3. 不要只问不记录
4. 不要让素材库成为摆设
